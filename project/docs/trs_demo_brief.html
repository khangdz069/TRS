TRS Rebuild - Demo hệ thống gợi ý testcase

Hệ thống hỗ trợ sinh viên nộp code C++, tự động chấm testcase và gợi ý một số testcase sai có giá trị học tập để sinh viên tập trung sửa lỗi.

1. Tổng quan

Project này là một hệ thống web hỗ trợ sinh viên nộp bài lập trình C++. Sau khi grader chấm bài và trả về danh sách testcase đúng/sai, backend gọi recommendation engine.

2. Ý tưởng chính

Phần mềm

Có frontend, backend, grader, database và các luồng người dùng teacher/student.

Mục tiêu

Không đưa toàn bộ lỗi cùng lúc, mà chọn tối đa 3 testcase có ích hơn.

3. Kiến trúc

Thành phần Vai trò
Frontend Giao diện cho giảng viên và sinh viên.
Backend Điều phối nghiệp vụ, lưu dữ liệu, gọi grader và recommendation.
Grader Compile code C++ và chạy bộ testcase.
Recommendation engine Xếp hạng testcase sai và chọn tối đa 3 testcase để gợi ý.

4. Luồng

1
Teacher tạo bài Tạo assignment và import danh sách sinh viên.
2
Student nộp bài Upload file zip chứa code C++.
3
Backend nhận Lưu submission và gửi sang grader.
4
Grader chấm Compile và chạy testcase.
5
Gợi ý Chọn testcase fail nên xem trước.
6
Feedback Sinh viên đánh giá mức hữu ích.

5. Model Python dùng như thế nào?

Backend Java kiểm tra rule nghiệp vụ, rồi gọi model Python để chọn group, chọn RSVD/timeSVD/LSTM và rank testcase fail.

Nhóm Model Cách hiểu khi demo
apr1 RSVD Dùng matrix RSVD để xếp hạng testcase fail.
apr2 timeSVD Dùng matrix timeSVD cho nhóm sinh viên này.
apr3 LSTM Dùng matrix LSTM cho nhóm sinh viên này.
unknown RSVD Không xác định được nhóm thì model service fallback về RSVD.
